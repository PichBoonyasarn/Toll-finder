1　Startup & Configuration Check

  • Run npm start in the folder C:\Users\Staff\Pich\work\Coding\toll-finder (or double-click start.bat)

2　Route Search (Basic Behavior)

With expressway — Destination: Tokyo Station

  • A blue route line appears on the map
  • The expressway entrance IC name is shown (e.g. Oyamazaki IC / Nagaokakyo IC)
  • The expressway exit IC name is shown (e.g. Tokyo IC / Shuto Expressway exit)
  • Distance shows a reasonable value (approx. 450–550 km)
  • Travel time shows a reasonable value (approx. 4–5 hours)
  • ETC toll fee is shown in orange with a ¥ amount
  • Route chips show the expressway names used (e.g. Meishin, Shin-Tomei)

Without expressway — Origin: Nagaokakyo City　Destination: Muko City

  • ETC fee: shows "No expressway (free)"
  • Entrance IC / Exit IC: shows "Not detected"

📍 Location Button

  • Click 📍 → the browser requests location permission
  • After granting permission, the field auto-fills with the current address

Error Handling

  • Enter an invalid destination (e.g. "aaaaaaaaa") → a red error message appears (no crash)
  • Search with an empty origin → shows "Please enter an origin" error

3　Document Scan — File Upload

Drop Zone UI

  • A drop zone appears below the destination field
  • Dragging a file over the zone turns the border green
  • Dragging away from the zone restores the border
  • Clicking the "Select file" link → opens the file picker (PDF / DOCX / TXT supported)
  • Clicking anywhere in the drop zone → opens the file picker

4　Coordinate Selection → Route Search (Integration Test)CRITICAL

Upload a file containing coordinates, then test in this order.
  • Click a coordinate card → the destination field fills with "35.xxxxx, 139.xxxxx"
  • The clicked card gets a green highlight
  • Click Search Route → the route is calculated correctly (skips geocoding, uses the coordinates directly)
  • The map and result cards (IC names, distance, time, ETC fee) display correctly

Coordinate Clear Check

  • After selecting a coordinate, manually edit the destination field → the selected coordinate resets (next search uses geocoding)
  • Click a different coordinate card → the previous highlight clears and the new card is highlighted

5　Edge Cases

  • Upload a second file → the picker updates to the new file's results (previous results disappear)
  • Upload a file containing the same coordinate twice → duplicates are removed, only 1 entry is shown
  • Upload a file with 10+ coordinates → a maximum of 10 entries are shown
